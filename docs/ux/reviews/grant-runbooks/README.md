# Grant Runbooks: Faculty Workflows

**Purpose:** Each faculty persona describes how they would use OpenResearchDataPlanner to plan a specific grant.  Not an exploratory review — a prospective runbook.  "Here's what I'd actually click, in order, and what I need out of it."

**Format:** One file per persona, one runbook per grant.  Steps follow the wizard sequence:
Welcome → Tier → (Questionnaire) → (Consultation) → Help Me Estimate → Grant Period → Services → Software → Retention → Results

**Use cases:**
- Input for IVaC video scripts (real scenarios, authentic voices)
- Regression testing reference (does each path produce the expected output?)
- Onboarding material (show new faculty "here's what someone like you would do")
- Design feedback source (where do researchers stall or guess?)

**How these were generated:** Sonnet, no extended thinking.  Persona-based simulation, not reasoning chains.
